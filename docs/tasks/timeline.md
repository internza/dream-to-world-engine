# Timeline

This timeline shows the progression of work during the Fall term. Each entry represents the major focus of the week and the deliverables completed for the project.

## Week 1
Set up the initial project direction. Defined the main concept of the Dream to World Engine and identified the core goal of converting natural language into three dimensional scenes.

## Week 2
Wrote the first version of the project description. Outlined the pipeline from prompt to world model and noted technical challenges to research.

## Week 3
Created and refined user stories to describe how different users interact with the system. Identified the main actors and their goals.

## Week 4
Produced the Level 0 system diagram. Represented the high level flow of user input, processing, and rendering.

## Week 5
Expanded the design with Level 1 and Level 2 diagrams. Added detail to system components and interactions between modules.

## Week 6
Worked on research related to scene graphs, object placement, procedural generation methods, and rendering structures.

## Week 7
Prepared the task list and effort planning for the rest of the term. Broke large work items into smaller actionable pieces.

## Week 8
Revised all diagrams and user stories based on feedback. Finalized the updated versions for the Fall design report.

## Week 9
Wrote the ABET concerns essay and self assessment. Reviewed all documentation for consistency with earlier assignments.

## Week 10
Built the full README table of contents. Organized the project files into the docs folder with clear structure for submission.

## Week 11
Completed revisions on all report sections. Finalized the Fall design report and ensured all links and documents were accurate.

This timeline reflects the steady development of the system and the core design work completed during the term.
