# Meeting Notes

These notes summarize work sessions, planning discussions, and design decisions made throughout the Fall term. Each entry reflects a distinct checkpoint where goals were set and progress was reviewed.

## Week 1
Reviewed initial project ideas and defined the core vision for the Dream to World Engine. Decided that the main goal for the term would be designing the pipeline from text to world generation.

## Week 2
Planned the structure for the project description and identified the main components required for the system. Chose to focus early work on input processing and scene representation.

## Week 3
Met to review early user stories. Adjusted several stories to better match realistic user actions. Chose to center all stories around clarity, speed, and intuitive world creation.

## Week 4
Worked through the first draft of the Level 0 diagram. Clarified system boundaries and reorganized components so the diagram follows a clear flow from input to rendering.

## Week 5
Completed detailed discussions for Level 1 and Level 2 diagrams. Decided on naming conventions for modules and refined the internal flow between systems.

## Week 6
Focused on research. Reviewed approaches for scene graphs, model hierarchies, and procedural placement. Captured notes on how these ideas can shape the next phase of development.

## Week 7
Outlined the first draft of the task list. Set priorities for the remainder of the term and planned the first timeline structure.

## Week 8
Reviewed all diagrams and identified which ones needed updates for clarity. Rewrote several labels and reorganized sections of the user stories.

## Week 9
Met to finalize writing tasks. Completed the ABET concerns essay and the self assessment. Prepared these sections for integration into the main report.

## Week 10
Organized the repository structure. Discussed the layout for the README and verified which files needed to move into the docs folder.

## Week 11
Completed final checks of all report materials. Ensured the table of contents worked correctly with all links. Prepared the final submission package.

These notes show the structure and direction of work sessions throughout the term.
